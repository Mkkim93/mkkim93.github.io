Introduction

안녕하세요. Java/Spring 기반 백엔드 개발자 김민규입니다.


가맹점 관리 시스템(백오피스)을 개발하면서, 레거시 코드를 점진적으로 개선하고 유지보수 가능한 구조로 리팩토링하는 일에 큰 보람을 느끼고 있습니다.


"왜 이렇게 되어 있을까?"를 항상 먼저 생각하고, 비즈니스 구현 정책을 결정할 때 항상 트레이드오프를 고려하여, 현 시점에서의 기술적 제약과 서비스 확장성을 함께 판단하여 합리적인 선택을 지향합니다.


단순히 동작하는 코드가 아닌, 팀 커뮤니케이션 비용을 최소화를 위한 코드를 지향합니다.

Values

Readable Code
  • 명확한 네이밍과 일관된 구조
  • 불필요한 추상화 지양
  • 코드 리뷰를 통한 품질 향상
Progressive Improvement
  • 빅뱅 리팩토링보다 점진적 개선
  • 기존 동작 보장하면서 구조 변경
  • 공통화를 통한 중복 제거
Problem Solving
  • 근본 원인을 파악하는 디버깅
  • 트러블슈팅 기록 습관
  • 팀원과의 지식 공유
Communication
  • 기술 결정에 대한 명확한 근거 제시
  • 비개발 직군과의 원활한 소통
  • 문서화를 통한 맥락 전달

How I Work

work-style.txt
// 1. 현재 코드를 충분히 이해한다
read("기존 코드");

// 2. 왜 이렇게 작성되었는지 맥락을 파악한다
analyze("히스토리", "비즈니스 요구사항");

// 3. 더 나은 방향을 설계한다
plan("점진적 개선안");

// 4. 기존 동작을 보장하면서 변경한다
refactor("안전하게");

// 5. 기록을 남긴다
document("다음 사람을 위해");
